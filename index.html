GoGetter — Queue Management Platform

Production-grade cloud-native slot-based queue management system. I owned the entire infrastructure layer, database administration, and all server-to-server connectivity across both regions.

GoGetter — Slot Based Queue Management System
PostgreSQL 18.3 AWS EC2 Next.js 14 Rocky Linux Nginx PM2 Streaming Replication DC · DR

In modern urban life, one of the most common yet underestimated frustrations is waiting in queues. Whether it is at a barbershop, a doctor's clinic, a vegetable vendor, or any neighbourhood service provider, people are routinely forced to stand in physical queues without any certainty of their waiting time. GoGetter is a full-stack web application that eliminates this problem by digitising queue management for local service providers — enabling customers to book slots, pay securely in advance, receive a live queue position, and track their turn in real time, all from a mobile-friendly browser without any app installation.

Problem & solution
Problems
No way to know wait time before physically travelling to a local provider
Unmanageable physical queues at peak hours — overcrowding and provider stress
No-shows after verbal bookings cause direct revenue loss and idle capacity
No affordable digital platform exists for small local businesses in India
Cash-only payments let customers walk away with zero financial accountability
Solutions
Live queue tracker shows exact position updated every 10 seconds automatically
Dynamic slot generation with atomic capacity enforcement via SELECT FOR UPDATE
Escrow wallet model — payment held by platform until service delivery confirmed
5% commission-only model — zero upfront subscription fee for providers
Auto-refund on rejection or cancellation within the same database transaction

Core features
⚡

Dynamic slot booking

Pure-JS deterministic slot generator. Atomic capacity enforcement via PostgreSQL SELECT FOR UPDATE prevents any overbooking under concurrent load.

🔒

Escrow wallet payments

Funds held by platform until service completion. 5% commission auto-deducted on release. Instant refunds on rejection — all in a single DB transaction.

📡

Live queue tracker

10-second polling delivers exact position, batch number, row within batch, and estimated wait time. Visual states: Waiting, Next Batch, Being Served Now.

🗂️

Provider dashboard

8-tab management interface — slots, bookings, live queue, wallet, services, history, analytics, and business settings. Online / Busy / Offline toggle in real time.

🛡️

Admin governance panel

Full provider lifecycle: approve, reject, suspend, reinstate, warn, delete. Force-complete bookings, manual refunds, and full audit log exportable as CSV.

🗄️

HA database cluster

Primary + Read Replica + PITR Backup in Mumbai DC. Full mirror in Singapore DR. promote_dr.sh enables sub-5-minute failover.

Slot generation algorithm
Pure JavaScript · No DB calls · Deterministic — The generateSlots(openTime, closeTime, slotDurationMins) utility runs identically on both the server (API validation) and the client (slot grid UI). A provider open 09:00–21:00 with 30-minute slots generates exactly 24 slots (09:00 to 20:30). Same inputs always produce the same output on both sides — the slot preview a provider sees during registration is guaranteed identical to what customers see on the booking page.
